# 05 · 前端观测台与部署形态

> 回答:项目有前端吗?跑在哪?Mac 怎么访问?
> 结论:核心是**无人值守后台引擎**,配一个**轻量 Web 观测台**作大屏+演示+审批入口。前端打包进 jar,单服务部署,公网 IP 直连访问。

## 一、定位:它本质是后台服务

现有 Argus 和 AIOps 改造,本质都是 **Spring Boot 后端服务**,跑服务器上,暴露 HTTP/SSE。

- **自愈闭环本身不需要前端** —— 告警进→自动诊断→自动处置,无人坐屏幕前点按钮。这是项目特点(真实 AIOps 核心都是后台引擎)。
- **但演示 + 高危审批需要界面**:
  - 演示:面试不能看 curl/Mongo 命令行;SSE 流式诊断天生适合可视化,是 SSE 技术点最佳展示载体
  - 改代码审批门(见 `04` §二):人在哪 review → 一个待审批页面

## 二、做什么:轻量 Web 观测台(不是客户端 App)

**定位:运维大屏 + 演示窗口 + 审批入口。不是"操作系统",别过度设计。**

| 区块 | 内容 | 展示的技术点 |
|---|---|---|
| 告警列表 | 实时告警,收敛后的 | 收敛/降噪可视化 |
| **诊断流** | 点告警,SSE 逐阶段流式渲染诊断过程 | **SSE 核心展示** |
| 自治面板 | 动作是自动执行还是等审批,置信度多少 | 分级自治 + 五道闸 |
| 待审批 | 高危动作(改代码)列此,人点批准/拒绝 | 审批门落地 |
| 案例库 | memory 沉淀的历史案例 | memory 闭环可视化 |

## 三、技术选型:别上重型框架

- **Vue3 单页 + `EventSource` 接 SSE**,够了
- 理由:后端岗,前端是展示层不是考点;炫前端反而主次不分;工作量半天~一天
- 简历叙事:"我做了诊断引擎,配了观测台",不是"我做了前端项目"
- **前端打包进 Spring Boot `src/main/resources/static/`,随 jar 一起发布**,不另起 Node 服务、不前后端分离部署。`docker-compose up` 一把起。

## 四、部署形态

```
┌─────────────────────────────────────┐
│  浏览器(Mac):Vue 观测台 EventSource │
└──────────────┬──────────────────────┘
               │ HTTP + SSE  (公网 IP:8080)
┌──────────────▼──────────────────────┐
│  Spring Boot 服务(jar / Docker)     │  ← 核心,跑服务器
│  ├ 告警接入 / 诊断引擎 / 自治处置      │
│  └ static/ 托管前端(打包进 jar)      │
└──────┬───────────┬───────────┬───────┘
   Redis        MongoDB      Milvus(prod)
```

WebFlux(Netty)默认绑 `0.0.0.0`,通常不用改:
```yaml
server:
  address: 0.0.0.0
  port: 8080
```

## 五、Mac 如何访问(主方案:公网 IP 直连)

**主方案 —— 公网 IP 直连(演示首选,一个 URL 甩给面试官即可)**
```
Mac 浏览器:  http://<服务器公网IP>:8080
```
前提(部署时必做):
- 云服务器**安全组/防火墙放行 8080 入站**(阿里云/腾讯云/AWS 控制台加一条入站规则)
- 服务监听 `0.0.0.0`(默认即是)
- 建议加一层基础保护:演示用 token / basic auth,避免公网裸奔(见 §六安全)

**备选 —— SSH 端口转发(无公网或想更安全时)**
```bash
ssh -L 8080:localhost:8080 user@<服务器IP>
# 然后 Mac 浏览器开 http://localhost:8080
```
不暴露公网端口,流量走 SSH 隧道。

## 六、两个必避的坑

**1. 公网暴露的安全**(security)
公网直连 8080 = 任何人扫到都能访问你的诊断接口。demo 也要兜底:
- 最少加一个静态 token 校验(请求头带 `X-Demo-Token`),或 Spring Security basic auth
- 别把真实 LLM API key / Mongo 凭据暴露在前端或公网可读配置里
- 演示完关掉安全组规则或停服务,别长期裸奔

**2. SSE 经反向代理被缓冲**
若前面挂 Nginx,必须关 SSE 缓冲,否则诊断流被攒着一次性返回:
```nginx
location / {
    proxy_pass http://localhost:8080;
    proxy_buffering off;            # SSE 必须关
    proxy_http_version 1.1;
    proxy_set_header Connection '';
}
```
demo 直连不挂 Nginx 则无此问题。

## 七、简历定位

- **核心卖点是后台无人值守诊断引擎**,观测台是配套,别本末倒置
- 可写:"提供 WebFlux SSE 流式诊断,前端 EventSource 实时渲染诊断全过程;观测台支持告警收敛展示、分级自治状态、高危动作人工审批"
- 别写成前端项目;别吹"全栈"——你是后端,观测台是展示层
